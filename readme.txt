Просмотр и изменение пользователей 

Приложение должно состоять из следующих частей:
1. База данных на любой СУБД
2. Серверная часть приложения на любом языке (java, c#, nodeJs или др.)
3. WEB Интерфейс отображения информации.

Требования к компонентам.

1. В базе данных должна храниться информация о пользователях и их контактах, а также
сопутствующие справочники (при необходимости).
Обязательные атрибуты пользователя 
1) Id
2) ФИО
3) Логин
4) Пароль
5) Дата рождения
6) Дата приема на работу
7) ИНН
8) Зар.плата 

Создайте не менее 15 пользователей, наполнение атрибутов – произвольно
Также у каждого пользователя есть «контакты». Контактные данные должны содержать:
1) Мобильный номер (1 или более)
2) Email (1 или более)

2. Web интерфейс
2.1. Создайте страницу для просмотра всех пользователей. Просто вывести в таблице список всех заведенных пользователей. По каждому пользователю показать все поля из базы данных + сколько полных лет пользователю

На странице реализовать фильтр:
По стажу сотрудников (в поле ввода вводится количество дней и необходимо отразить только тех сотрудников, которые проработали меньше дней, чем введенное число)

2.2. Создайте страницу для заведения нового пользователя и изменения данных о существующих пользователях
